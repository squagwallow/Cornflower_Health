### 1. Root cause found and fixed — `invalid_request_url` on block appends
The Perplexity Computer session's deploy failures were caused by a **single wrong HTTP verb**.
`api_append_children` called `client.post(…/blocks/{id}/children)` but the Notion API
requires **PATCH** for that endpoint. `POST` returns `400 invalid_request_url`; `PATCH` returns `200`.

- `api_create_page` was passing `title` at top level instead of `properties.title.title[…]`
- `build_trends_blocks` used `tab_block`/`tab` — not valid in API version 2022-06-28
- `build_settings_blocks` had Python 3 surrogate char literals (`\ud83d\udfe2` etc.)

## Notion Views — What They Are and How to Add Them

A "linked database view" embeds a live, filtered slice of the Daily Health Metrics
database directly inside a dashboard page. Think of it as a dynamic table or card
grid that always shows current data based on filters you set.

**Where they're needed (gray placeholder text marks each spot):**

| Page | Section | What to add | Filter |
|------|---------|-------------|--------|
| Daily Dashboard | Exertion & Stress | Linked view, gallery type | date = today |
| Daily Dashboard | Notes & Log | Linked view, table type | date = today, show: meds_notes, notes |
| Daily Dashboard | Workout Log | Linked view, table type | date = today, show: workout fields |
| Trends | Each 5/10/20/40d section | Linked view, table type | date is in last N days |
| Flags & Alerts | Top section | Linked view, table type | any flag = true, last 14 days |
| Full Data Table | Main section | Linked view, table type | no filter (show all) |

**How to add one in Notion:**
1. Open the dashboard page
2. Click on the gray placeholder text line, then delete it
3. Click the `+` block button → search "Linked view" → select **Linked view of database**
4. Select **Daily Health Metrics**
5. Notion will create a default table view — click the view name to configure:
   - Set filters (e.g., Date = Today, or Date is in the last 7 days)
   - Hide columns you don't need (click Properties → toggle off)
6. Rename the view to match the section
